# Completed Plans

Completed plans are filed here by category and indexed in `index.md`. The default filing location is `project/plans-completed/`; use `project/changelog/plans/` only when explicitly requested.

## Categories

- `implementation/` — Feature or code implementation plans
- `investigation/` — Research, diagnosis, or discovery plans
- `migration/` — Data, architecture, or repository migrations
- `review/` — Review, audit, or validation plans
- `tooling/` — Tooling, workflow, or development-environment plans

## Filing a completed plan

Move the plan from `project/plans/` or `project/build/` into the appropriate category, prefixing the filename with `YYYY-MM-DD-` when needed. Then add a row to this index and a `Type=plan` row to `project/changelog/index.md` with a File path such as `../plans-completed/implementation/YYYY-MM-DD-plan.md`.
