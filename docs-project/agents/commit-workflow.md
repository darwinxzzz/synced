# Commit & Pull Request Workflow

- Follow Conventional Commit-style prefixes such as `feat:`, `fix:`, `test:`, `docs:`, and `refactor:`.
- Keep commits focused and make them from the repository that owns the changed files.
- Pull requests should explain the behavior change, list verification commands, link the relevant issue or plan, and include screenshots or recordings for UI changes.
- Call out migrations, seed data, and environment changes explicitly.
- Main-repository changes are committed from the project root; Workflow-Scripts changes are committed from `Workflow-Scripts/`.
