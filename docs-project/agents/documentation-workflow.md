# Documentation Workflow

- Long-lived project documentation belongs in `docs-project/`.
- Agent-facing detailed guidance belongs in `docs-project/agents/`.
- Existing application documentation in `docs/` is retained and remains valid for application-specific architecture and assets.
- Project metadata, active plans, changelog entries, and troubleshooting entries belong under `project/`.
- Update documentation when a workflow, command, architecture boundary, security rule, or user-facing behavior changes.
- Keep root agent files slim and link to detailed documentation from `docs-project/agents/`.
