## Issues Encountered & Fixed

### 1. ESM Compatibility (`__dirname` not defined)

**Problem:** Project has `"type": "module"` in `package.json`, so Node.js ESM doesn't define `__dirname` globally.

**Fix applied to all Playwright files:**
```typescript
import { fileURLToPath } from "url";
const __dirname = path.dirname(fileURLToPath(import.meta.url));
```

### 2. Test Timeout Too Short for MFA

**Problem:** Playwright default test timeout is 30,000ms. The MFA modal appeared correctly but the user needed more time to read the terminal prompt and enter their 6-digit code.

**Fix:** Added `test.setTimeout(180_000)` (3 minutes) inside both setup test bodies.

### 3. Member Credentials Had Admin Role

**Problem:** The credentials saved as `TEST_MEMBER_EMAIL` in `.env.test` belong to an admin account — after MFA, the app redirected to `/admin/dashboard` instead of `/member/dashboard`, causing the setup to time out.

**Fix:** Updated `member.setup.ts` to accept either dashboard URL:
```typescript
await page.waitForURL("**/*dashboard*", { timeout: 120_000 });
```
The session is saved regardless of role, with a note logged to console.

## MFA + Localhost — Confirmed Working

**Concern raised:** Does Supabase MFA work with a locally hosted app?

**Answer:** Yes — confirmed working. The MFA modal appeared in the browser during the test run. TOTP verification happens between the browser and Supabase's cloud API (`https://[project].supabase.co`). The local Next.js dev server is not involved in TOTP verification at all.
