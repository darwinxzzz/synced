## What We Discovered

### Rate Limiting
- **Library used**: None — custom hand-rolled in-memory implementation using a `Map`
- **Location**: `src/middleware.ts` lines 4–17
- **Limits**: Auth routes (`/auth`, `/login`) = 10 req/60s; tRPC routes (`/api/trpc`) = 100 req/60s
- **Known limitation**: In-memory only — resets on restart, breaks across multiple server instances
- **Recommended upgrade**: [Upstash Redis](https://upstash.com/) with `@upstash/ratelimit` for distributed rate limiting

## Problems Found & Fixed

### Problem 1 — Redundant `profiles` DB queries in tRPC (FIXED)

**File**: `src/server/api/trpc.ts`

**Before**: Profile was fetched inside both `protectedProcedure` and `adminProcedure` separately — 2 DB hits per protected tRPC call on top of `getUser()`.

**After**: Profile is fetched once in `createTRPCContext` and stored as `ctx.profile`. Both procedures now read from context — 0 extra DB calls.

```ts
// createTRPCContext now fetches profile once
const profile = user
  ? (await supabase.from("profiles").select("role, status").eq("id", user.id).single()).data
  : null;
return { supabase, user, profile, ...opts };
```

### Problem 2 — `/api/auth/departments` had wrong auth + 3 DB calls (FIXED)

**File**: `src/app/api/auth/departments/route.ts`

**Before**:
1. `getUser()` call
2. `profiles` query to check role/status (admin gate)
3. Second `profiles` query for actual departments
→ Returned `[]` for non-admin users — **signup form dropdown was broken** for all regular users, silently falling back to `DEPARTMENT_FALLBACK`

**After**:
1. Single `profiles` query for departments (no auth check)
→ Department names are not sensitive — public list is correct behavior for signup form


```ts
// src/middleware.ts replacement
import { Ratelimit } from "@upstash/ratelimit"
import { Redis } from "@upstash/redis"

const authLimiter = new Ratelimit({
  redis: Redis.fromEnv(),
  limiter: Ratelimit.slidingWindow(10, "60 s"),

  redis: Redis.fromEnv(),
  limiter: Ratelimit.slidingWindow(100, "60 s"),
```

### 3. `React cache()` for profile deduplication within render (Low effort)
If Server Components ever call `getProfile` separately from tRPC context, wrap it in `React cache()` to deduplicate within a single render pass.

```ts
// src/lib/supabase/profile.ts (new file)
import { cache } from 'react'
import { createClient } from '~/lib/supabase/server'

export const getCachedProfile = cache(async (userId: string) => {
  const supabase = await createClient()
  const { data } = await supabase
    .from('profiles')
    .select('role, status')
    .eq('id', userId)
    .single()
  return data
})
```
